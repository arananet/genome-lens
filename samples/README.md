# Synthetic test samples

Three genome files for testing genome-lens. **None of these files contain real
personal data.** They represent a fully fictional person invented for functional
testing. rsids in the range `rs1000000000+`, `rs2000000000+`, and `rs3000000000+`
are fabricated filler variants — not real SNPs.

## Files

| File | Format | SNPs | Notes |
|---|---|---|---|
| `synthetic-23andme.txt` | 23andMe raw export | 2,517 | Array-based, GG genotype style |
| `synthetic-ancestry.txt` | AncestryDNA raw export | 1,817 | Split allele1/allele2 columns |
| `synthetic-myheritage.txt` | MyHeritage Low-pass WGS | 2,217 | Quoted CSV, ~2% no-calls |

## Usage

Upload any of these files to the genome-lens app to test the full pipeline
without using real personal genomic data. The 23andMe file is a good default.

## Generation

Files were generated by `scripts/generate-samples.py` using Python's `random`
module with fixed seeds (42 / 99 / 77) for reproducibility.
All positions are GRCh37 coordinates from public databases (NCBI dbSNP).
Filler variants use sequential synthetic rsids and random positions drawn
uniformly from each chromosome's length range.
